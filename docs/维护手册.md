# 维护手册

## 1. 日志路径

- 应用日志：`/www/wwwroot/cs/logs/app.log`
- Nginx 访问日志：宝塔站点默认日志目录
- Nginx 错误日志：宝塔站点默认日志目录
- PHP 错误日志：对应 PHP 版本 `log` 目录
- 任务状态文件：`/www/wwwroot/cs/data/tasks/*.json`
- ZIP 缓存目录：`/www/wwwroot/cs/tmp/`

## 2. 备份策略

建议分三层：

1. 代码备份：每日备份 `/www/wwwroot/cs`，保留 7 个版本。
2. 数据备份：重点备份 `/www/wwwroot/cs/1`，建议同步至对象存储或异地机器。
3. 配置备份：导出宝塔站点配置、Nginx 配置、SSL 证书与计划任务配置。

建议命令：

```bash
tar -czf /backup/cs-code-$(date +%F).tar.gz /www/wwwroot/cs
```

若源目录体积很大，建议单独使用 `rsync`：

```bash
rsync -av --delete /www/wwwroot/cs/1/ backup@server:/data/backup/cs-1/
```

## 3. 升级步骤

1. 备份当前代码与配置。
2. 在低峰期暂停访问或设置维护窗口。
3. 上传新版本文件到临时目录。
4. 对比并保留生产环境中的密码、环境变量、Nginx 配置。
5. 覆盖更新以下目录：
   - `index.php`
   - `download.php`
   - `api/`
   - `lib/`
   - `py/`
   - `static/`
   - `docs/`
6. 如 Python 依赖有变更，执行：

```bash
source /www/wwwroot/cs/py/venv/bin/activate
pip install -r requirements.txt
```

7. 重载服务：

```bash
bt reload
```

8. 执行上线验证：
   - 登录页可打开
   - 列表接口返回正常
   - 文件断点续传正常
   - 目录打包进度正常

## 4. 故障排查

### 4.1 无法登录

检查：

- `CS_APP_PASSWORD` 是否已配置
- Session 保存目录是否可写
- 登录失败次数是否已触发 10 分钟封禁
- 反向代理是否正确透传真实 IP

### 4.2 目录列表为空

检查：

- `CS_SOURCE_ROOT` 是否指向正确目录
- 源目录是否具备读取权限
- 目录下是否只有被系统过滤的敏感文件

### 4.3 ZIP 任务一直不动

检查：

- PHP 是否允许 `exec`
- `CS_PYTHON_BIN` 是否存在
- `py/venv` 是否安装 `tqdm`
- `data/tasks/` 与 `tmp/` 是否可写
- `logs/app.log` 是否存在 Python 启动异常

### 4.4 大文件续传失败

检查：

- Nginx 是否保留 `Accept-Ranges`
- CDN 是否拦截 `Range` 请求头
- 浏览器或下载器是否发送合法 `Range` 头

## 5. 性能维护建议

- `tmp/` 目录保持每日清理，避免磁盘堆积
- 大量小文件打包时优先使用 SSD
- 如并发任务较多，可将 `tmp/` 与 `data/tasks/` 放在独立高速磁盘
- 建议通过 `htop`、`iostat`、`vmstat` 观察 CPU、IO、内存
- 20 并发 ZIP 压缩场景建议启用 4 核以上 CPU

## 6. 安全维护建议

- 定期轮换 `CS_APP_PASSWORD`
- 全站启用 HTTPS
- 给站点增加 Fail2ban 或 WAF 规则
- 限制 SSH 管理 IP
- 定期清理 `tmp/`、日志和历史任务 JSON

## 7. 回滚步骤

1. 停止新版本访问。
2. 恢复上一版代码包。
3. 保留生产环境中的 `/www/wwwroot/cs/1` 数据目录不变。
4. 重载 `nginx` 与 `php-fpm`。
5. 再次执行功能验证。

## 8. 定期巡检清单

- 证书是否在 15 天内到期
- `tmp/` 是否存在超过 24 小时未清理的 ZIP
- `logs/app.log` 是否存在高频错误
- 宝塔计划任务是否按时执行
- `data/tasks/` 是否有长期停留在 `running` 状态的任务文件
